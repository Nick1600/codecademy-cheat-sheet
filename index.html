Why CSS Grid is More Effective Than HTML Tables

In my limited experience with CSS Grid, and even more limited experience with HTML Tables, I find that in almost every way, the Grid system is more powerful, versatile, and user friendly than HTML tables.

The Humble HTML Table

HTML tables are a reasonable easy way to convey information in a table layout using just HTML.There is some functionality that allows customization of the table, but it is pretty limited. You can set the amount of rows, and columns, as well as adding cells that span multiple rows or columns, but that is about the extent of it. Of course, there is pretty limitless customization with CSS.

HTML Table Cheat-Sheet

This is a table containing HTML table syntax, with a brief description alongside. You will notice that this table looks much more put together, and is easer to read due to some extra CSS.

Syntax description
>table> >/table> The opening and closing table tag
>thead> >/thead> Specifies the head of the table.
>tbody> >/tbody> Specifies the body, or main section of the table.
>tfoot> >/tfoot> Specifies the footer section of the table.
>tr> >/tr> Creates a new row in the table.
>td> >/td> Creates a 'cell' in the table, adding columns across the rows.
>td colspan="2"> >/td> The new cell created will span across two columns.
>td rowspan="2" >/td> The new cell will span down two rows.
That's the long and short of HTML table syntax!

Conclusion

Hopefully this helps with understanding HTML tables. We have looked at some of the limitations of HTML tables, and also gone over most of the syntax involved.

The most compelling argument for HTML tables is their relative ease> However, I would say that the biggest drawback the humble table faces is their non-responsive nature. This is where CSS Grids truly shine strong. But thats a story for another time.
